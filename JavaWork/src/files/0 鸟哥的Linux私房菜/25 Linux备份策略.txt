具有备份意义的文件通常可以分为两大类
1，系统基本设置信息	2，类似网络服务的内容数据
操作系统本身需要备份的文件有哪些？
/etc整个目录	/home整个目录	/var/spool/mail	/boot	/root	如果你自行安装过软件，那么/usr/local和/opt需要备份
网络服务的数据库方面哪些文件需要备份？
各种数据库文件	MYSQL：/var/lib/mysql
推荐至少需要备份的数据
/boot /etc /home /root /usr/local /var
不需要备份的数据
/dev /proc /mnt或/media /tmp
备份有哪些方式？
增量备份	差异备份
系统进行完第一次完整备份后，经过一段时间的运行，比较系统与备份文件之间的差异，仅仅备份有差异的文件。
增量备份
完整备份常用的工具
dd cpio dump/restore
可以直接进行增量备份的是什么软件
dump
差异备份和增量备份相似，第一次都需要什么？
完整备份
进行镜像备份的命令
rsync
差异备份和增量备份的不同点
磁盘容量大，还原快
备份一些关键数据，可以使用什么命令
tar
每日备份和每周分别备份哪些数据？
每日备份：MYSQL数据库 每周备份：/home /var /etc /boot /usr/local等
每日系统备份的Shell Script
每周系统备份的Shell Script
远程备份使用什么工具
FTP或rsync
